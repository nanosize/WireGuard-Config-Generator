Privacy Policy

1. Key Generation

We use the open-source libsodium (TweetNaCl) cryptographic library to generate high-security Curve25519 key pairs.

Both public and private keys are 32 bytes (256 bits) long, ensuring resistance against current cryptographic attacks.

When a pre-shared key (PSK) is needed, it is generated via the library’s cryptographically secure random number generator (nacl.randomBytes(32)), adding an extra layer of protection.

2. Generation Location (Server vs. Client)

By default, key pairs and PSKs are generated server-side in our Node.js environment (src/configGenerator.js).

Configuration files (server.conf and peer-*.conf) are assembled in memory, zipped, and delivered over HTTPS.

QR code rendering happens entirely in your browser; no private key material is ever sent back to our servers after download.

3. Data Protection & Secure Communication

  • HTTPS (TLS 1.2+): All data transfers are encrypted with TLS 1.2 or higher, preventing eavesdropping and tampering.
  • In-Memory Processing: Keys and configuration data are held only in memory and discarded immediately after delivery—nothing is written to disk or logs.
  • Secure RNG: All randomness is produced by a CSPRNG built into the cryptographic library, ensuring unpredictability.

4. Data Retention & Logging

  • No private keys, PSKs, or config data are ever persisted to databases or file systems.
  • Access and application logs do not record any key material or configuration details.
  • We do not collect or store any personal information such as names, email addresses, or IP addresses.

5. User Responsibilities

It is your responsibility to securely store downloaded private keys and PSKs. Do not share them with unauthorized parties.

If you lose a private key, it cannot be recovered—you must generate a new key pair and reconfigure your WireGuard peers.

6. Site Operations & Monitoring

  • We use Google Search Console to monitor search performance and indexing status.
  • All data collected via Google Search Console is anonymized and used solely for site health and improvement metrics.
