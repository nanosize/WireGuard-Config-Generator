Privacy Policy

1. Key Generation Mechanism

We use the TweetNaCl (libsodium) cryptographic library to generate Curve25519 key pairs, ensuring both high performance and strong security.

The generated public and private keys are 32 bytes (256 bits) long, making them computationally infeasible to break.

Pre-shared keys (PSKs), when needed, are generated as 32-byte random sequences via a cryptographically secure random number generator (nacl.randomBytes(32)), adding an extra layer of protection.

2. Generation Location (Server vs. Client)

By default, key pairs and PSKs are generated on our server environment (Node.js, in src/configGenerator.js). Once generated, configuration files (server.conf and peer-*.conf) are assembled in memory, zipped, and delivered to you.

QR code rendering is performed entirely in your browser. Your keys are never sent back to our servers after the initial download.

3. Data Protection and Security

  • Encrypted Communication: All data transfers occur over HTTPS (TLS), preventing eavesdropping and tampering.
  • In-Memory Handling: Keys are held only in server memory and are discarded immediately after ZIP delivery. No keys or configurations are written to disk or logs.
  • CSPRNG Usage: All random values are generated using a cryptographically secure pseudo-random number generator.

4. Data Retention and Logging

  • No private keys, PSKs, or configuration data are persisted in databases or file systems.
  • Access and application logs do not record any key material or configuration details.
  • We do not collect personal information such as names, email addresses, or IP addresses.

5. User Responsibility

You are responsible for securely storing your downloaded private keys and PSKs. Do not share them with unauthorized parties.

If you lose your private key, you must generate a new key pair and reconfigure your WireGuard peers, as lost keys cannot be recovered.
